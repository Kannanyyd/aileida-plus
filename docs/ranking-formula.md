# 性价比评分公式

## 综合分（0-100）

```
综合分 = 0.40 × 价格分 + 0.20 × 上下文分 + 0.20 × 能力分 + 0.10 × 稳定性分 + 0.10 × 可信度分
```

## 各维度计算

### 价格分

- 取输入 + 输出价的等权平均，在同类模型中做归一化
- 越低越好（min = 100, max = 0）
- 含缓存价的模型加权优化

### 上下文分

| 上下文 tokens | 评分 |
|---|---|
| ≥ 2M | 100 |
| ≥ 1M | 90 |
| ≥ 500K | 80 |
| ≥ 200K | 70 |
| ≥ 128K | 60 |
| ≥ 32K | 40 |
| ≥ 8K | 25 |
| < 8K | 10 |

### 能力分

| 能力 | 权重 |
|---|---|
| reasoning | 18 |
| vision | 18 |
| function-call | 16 |
| long-context | 16 |
| cache | 10 |
| audio | 8 |
| video | 8 |
| json-mode | 8 |

### 稳定性分

- MVP 阶段固定 70（后续接入历史价格波动率）

### 可信度分

- confidence_score × 100
