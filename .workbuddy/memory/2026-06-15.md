## Local changes implemented

- `apps/web/lib/db/queries.ts`
  - Added derived freshness fields to model records:
    - `source_checked_at`
    - `pricing_checked_at`
    - `official_source_checked_at`
    - `freshness_status`
    - `source_age_hours`
    - `pricing_age_hours`
    - `model_age_days`
  - Added derived supersession fields:
    - `has_newer_family_model`
    - `superseded_by_model_id`
    - `is_current_default_pick`
  - Enriches models from `source_fetch_logs`, `model_discovery_logs`, and `latest_model_candidates`.
  - Added `dataFreshnessOverview()` for homepage freshness cards.

- `apps/web/lib/rank/score.ts`
  - Adds `hideStale`, `hideSuperseded`, `maxSourceAgeHours`, `homepageStrict`.
  - Ranking filters stale and superseded models by default.
  - Homepage strict mode filters suspicious/manual-review/missing-source/low-confidence stale candidates.
  - Ranking API items now include freshness/supersession fields and `why_ranked`.

- `apps/web/app/api/v1/rankings/[type]/route.ts`
  - Added query params:
    - `max_source_age_hours`
    - `hide_stale`
    - `hide_superseded`

- `apps/web/app/api/v1/recommend/route.ts`
  - Default non-cheapest recommendations filter superseded models and stale data over 72h.
  - Recommendation payload includes freshness/supersession fields.

- `apps/web/app/page.tsx`
  - Homepage title/copy tightened into a more professional API pricing intelligence positioning.
  - Homepage Top8 now uses strict 12h freshness and current-main-model ranking.
  - Adds freshness cards for model discovery, pricing source, and CNY pricing updates.

- `apps/worker/src/cli/audit-freshness.ts`
  - New command for source/pricing freshness audit.

- Scripts:
  - root `audit:freshness`
  - root `freshness:audit`
  - root `crawl:pricing`
  - worker `audit:freshness`
  - worker `crawl:pricing`
